Create API route (route.ts):
- POST /api/onboarding/complete
- Use createClient from @/lib/supabase/server
- Get authenticated user (return 401 if not authenticated)
- Parse and validate request body with Zod schema:
  - preferences: timezone, sleep times, meal times, buffer, commute (all with proper types)
  - commitments: array of { title, dayOfWeek, startTime, endTime }
  - goals: array of { title, hoursPerWeek }
- Sanitize all string inputs using sanitize.ts
- Insert into user_preferences table (upsert - update if exists)
- Insert commitments into fixed_commitments table (delete existing first, then insert new)
- For goals: Store in a simple goals table (we'll expand in Phase 5) OR just store in user_preferences.initial_goals as JSONB for now
- Update profiles.onboarding_completed = true
- Return success response
- Wrap in try/catch with proper error responses
