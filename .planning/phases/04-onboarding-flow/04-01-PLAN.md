<task type="auto">
  <name>Task 1: Create onboarding database tables migration</name>
  <files>supabase/migrations/00003_onboarding_tables.sql</files>
  <action>
Create migration with three tables:

1. `user_preferences` table:
   - id: uuid primary key (default gen_random_uuid())
   - user_id: uuid references auth.users on delete cascade (unique)
   - timezone: text not null default 'America/New_York'
   - sleep_start: time not null default '23:00' (bedtime)
   - sleep_end: time not null default '07:00' (wake time)
   - meal_breakfast_start: time default '08:00'
   - meal_breakfast_duration: integer default 30 (minutes)
   - meal_lunch_start: time default '12:00'
   - meal_lunch_duration: integer default 45
   - meal_dinner_start: time default '18:30'
   - meal_dinner_duration: integer default 60
   - buffer_minutes: integer default 15 (transition buffer)
   - commute_morning_start: time (nullable - no commute)
   - commute_morning_duration: integer
   - commute_evening_start: time
   - commute_evening_duration: integer
   - created_at: timestamptz default now()
   - updated_at: timestamptz default now()

2. `fixed_commitments` table:
   - id: uuid primary key
   - user_id: uuid references auth.users on delete cascade
   - title: text not null
   - day_of_week: integer not null (0=Sunday, 6=Saturday)
   - start_time: time not null
   - end_time: time not null
   - is_recurring: boolean default true
   - created_at: timestamptz default now()
   - updated_at: timestamptz default now()

3. Enable RLS on both tables with User-Owns-Row pattern (SELECT, INSERT, UPDATE, DELETE).

4. Add updated_at trigger for both tables.

Follow patterns from docs/rls-policy-patterns.md exactly.
  </action>
