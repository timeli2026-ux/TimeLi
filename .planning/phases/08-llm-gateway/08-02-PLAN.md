  <name>Task 1: Create goal parser prompt template</name>
  <files>src/lib/llm/prompts/goal-parser.ts</files>
  <action>
Create prompt template for parsing natural language goals.

Export:
- buildGoalParserPrompt(): system prompt instructing LLM to extract structured goal
- parseGoalResponse(response: string): ParsedGoal | { needsClarification: true, questions: string[] }

Structured output format (JSON in code block):
```json
{
  "title": "string (concise goal name)",
  "description": "string (optional details)",
  "hours_per_week": number,
  "frequency": "daily" | "weekly" | "custom",
  "sessions_per_week": number,
  "preferred_duration_minutes": number,
  "realm_name": "health" | "career" | "learning" | "relationships" | "creativity" | "finance" | "personal" | "spiritual",
  "deadline": "YYYY-MM-DD" | null,
  "priority": "low" | "medium" | "high"
}
```

If LLM cannot determine required fields, it should return:
```json
{
  "needsClarification": true,
  "questions": ["What realm does this belong to?", "How many hours per week?"]
}
```

Prompt should emphasize:
- Extract only what's explicitly stated
- Ask for clarification on missing required fields
- Infer reasonable defaults for optional fields

<task type="auto">
  <name>Task 3: Create parse-goal API endpoint</name>
  <files>src/app/api/llm/parse-goal/route.ts</files>
  <action>
Create POST /api/llm/parse-goal endpoint.

Request body: { message: string, context?: string }
Response: { goal: ParsedGoal } or { needsClarification: true, questions: string[] }

Implementation:
1. Authenticate user (createClient + getUser)
2. Rate limit (10 requests per minute per user)
3. Check LLM availability (getLLMStatus)
4. Check cache (generateCacheKey + getCached) - return cached if hit
5. Build prompt using buildGoalParserPrompt
6. Call LLM via getLLMProvider
7. Parse response using parseGoalResponse
8. Record token usage (recordTokenUsage)
9. Cache response (setCached)
10. Return structured result

Error handling:
- 401 for unauthenticated
- 429 for rate limited
- 503 for LLM unavailable
- 500 for parse failures (log but don't expose details)
  </action>
