<objective>
Implement email/password authentication with Supabase Auth.

Purpose: Enable users to create accounts and sign in with email/password credentials, establishing the core authentication flow for TimeLi.
Output: Working signup and login pages, auth callback route, and Next.js middleware for route protection.
</objective>

<prerequisites>
This plan assumes Supabase credentials are configured via environment variables.

Required env vars (see `src/lib/env.ts`):
- `NEXT_PUBLIC_SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`

Optional (recommended for later server-side admin tasks, NOT needed for basic signup/login UI):
- `SUPABASE_SERVICE_ROLE_KEY` (server-only, never exposed to client)

Hosted Supabase (Option 2) setup checklist:
- In Supabase Dashboard → Settings → API, copy Project URL + `anon` key into your environment variables.
- In Supabase Dashboard → Authentication → URL Configuration:
  - Set **Site URL** to your app URL (local: `http://localhost:3000`, prod: your Vercel domain).
  - Add **Redirect URLs** including:
    - `http://localhost:3000/auth/callback`
    - `https://<your-vercel-domain>/auth/callback`
  - Ensure email confirmations redirect correctly to `/auth/callback`.

Vercel deployment checklist (to deploy publicly, safely):
- Add the same env vars in Vercel Project → Settings → Environment Variables:
  - `NEXT_PUBLIC_SUPABASE_URL`
  - `NEXT_PUBLIC_SUPABASE_ANON_KEY`
  - (Optional) `SUPABASE_SERVICE_ROLE_KEY` **ONLY** for Server/Edge environments, never used in client code
- Use separate Supabase projects (or at minimum separate keys) for **dev** and **production** if possible.
</prerequisites>

<task type="auto">

  <files>src/app/auth/callback/route.ts, src/middleware.ts</files>
  <action>
Auth callback route (src/app/auth/callback/route.ts):
- Handle GET request with code parameter from email verification
- Exchange code for session using supabase.auth.exchangeCodeForSession()
- Redirect to /dashboard (or / for now) on success
- Redirect to /login?error=auth on failure
- Use server Supabase client (createClient from @/lib/supabase/server)

Middleware (src/middleware.ts):
- Create middleware that refreshes auth session on each request
- Use createServerClient from @supabase/ssr with request/response cookie handling
- Apply to all routes except static files, _next, and public assets
- Protected routes pattern: routes starting with /dashboard require auth
- If accessing protected route without session, redirect to /login
- If accessing /login or /signup with session, redirect to /dashboard

Middleware config:
- matcher: exclude /_next/static, /_next/image, favicon.ico, and public folder
  </action>
