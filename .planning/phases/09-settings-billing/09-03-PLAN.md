<task type="auto">
  <name>Task 1: Create usage tracking service</name>
  <files>src/lib/services/usage-service.ts</files>
  <action>
Create usage tracking service with:
1. `getCurrentPeriod(userId: string)` - Returns current billing period (month start to month end) or creates one if missing
2. `incrementGenerations(userId: string)` - Increments schedule_generations, returns new count
3. `incrementRecalibrations(userId: string)` - Increments recalibrations, returns new count
4. `getUsage(userId: string)` - Returns current period usage with limits:
   ```typescript
   {
     generations: { used: number, limit: number },
     recalibrations: { used: number, limit: number },
     periodEnd: Date
   }
   ```
5. `checkCanGenerate(userId: string)` - Returns boolean, checks against USAGE_LIMIT_GENERATIONS
6. `checkCanRecalibrate(userId: string)` - Returns boolean, checks against USAGE_LIMIT_RECALIBRATIONS

Period is calendar month (1st to last day). Use env vars for limits (default 200/500).
Handle the case where user is on trial (no limits) vs active subscription (has limits).
  </action>
  <verify>Service functions compile without errors</verify>
  <done>Usage service tracks generations and recalibrations per period</done>
</task>

<task type="auto">
  <name>Task 2: Create usage API and integrate with schedule generation</name>
  <files>src/app/api/usage/route.ts, src/lib/scheduling/generate-schedule.ts</files>
  <action>
1. GET /api/usage:
   - Requires auth
   - Returns current usage stats from usage service
   - Include subscription status (trialing, active, etc.)

2. Integrate trial activation (BILL-03) into schedule generation:
   - In generateSchedule or the schedule API route, after successful generation:
   - Check if user has 'inactive' subscription status
   - If inactive, create subscription record with status='trialing', trial_start=now, trial_end=now+30 days
   - This means trial starts on first schedule generation, not on signup
   - Use incrementGenerations() to track usage

3. If user is over limit and not trialing:
   - Return 429 with helpful error message
   - Include current usage and limit in response

Note: Trial users have unlimited usage. Only check limits for 'active' subscriptions.
  </action>
  <verify>GET /api/usage returns usage stats, schedule generation increments counter and can trigger trial</verify>
  <done>Usage tracked on generation, trial starts on first generation</done>
</task>
