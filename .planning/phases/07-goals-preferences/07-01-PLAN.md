Create two API route files following existing patterns:

**src/app/api/goals/route.ts** (collection routes):
- GET: List all goals for authenticated user
  - Query user_goals with realm info via join
  - Return array of goals with realm name
  - Order by realm, then by title
- POST: Create new goal
  - Validate with goalFormSchema
  - Sanitize title with sanitizeUserInput
  - Insert into user_goals with user_id from session
  - Return created goal

**src/app/api/goals/[id]/route.ts** (single goal routes):
- GET: Get single goal by ID
  - Verify user owns goal (RLS handles this, but check result)
  - Return goal with realm info
- PUT: Update goal
  - Validate with goalFormSchema (partial)
  - Sanitize title
  - Update user_goals where id matches and user_id matches
  - Return updated goal
- DELETE: Delete goal (or archive by setting is_active=false)
  - Delete from user_goals where id matches and user_id matches
  - Return success

All routes:
- Use createClient from @/lib/supabase/server
- Check auth with getUser()
- Return 401 if not authenticated
- Return 404 if goal not found
- Return 400 for validation errors
- Use (supabase as any) for tables not in generated types

<task type="auto">
  <name>Task 3: Create migration endpoint for onboarding actions</name>
  <files>src/app/api/goals/migrate/route.ts</files>
  <action>
Create POST /api/goals/migrate endpoint that:

1. Reads user_preferences.initial_actions (JSONB) for authenticated user
2. Reads user_preferences.life_realms (JSONB) for realm mapping
3. For each initial_action:
   - Find matching realm in life_realms table (by name) or create it
   - Create user_goal with:
     - title from action.title
     - realm_id from matched/created realm
     - hours_per_week calculated from action.timesPerWeek * action.minutesPerSession / 60
     - Default values for other fields
4. Clear initial_actions JSONB after successful migration
5. Return count of migrated goals

This is a one-time migration endpoint called when user first visits /goals after onboarding.

Handle edge cases:
- Already migrated (initial_actions is null/empty): return {migrated: 0, alreadyMigrated: true}
- Partial realm matches: create missing realms in life_realms table
- Duplicate goals (same title in realm): skip with warning

Use transaction-like pattern: if any insert fails, don't clear initial_actions.
  </action>
  <verify>
Endpoint handles cases correctly:
- TypeScript compiles
- Returns 401 without auth
  </verify>
  <done>POST /api/goals/migrate converts JSONB initial_actions to user_goals table entries</done>
</task>
