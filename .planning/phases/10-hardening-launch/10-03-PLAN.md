<task type="auto">
  <name>Task 1: Audit external resources and add CSP header</name>
  <files>next.config.ts</files>
  <action>
    1. **Audit all external resources** used by the app:
       - Fonts: Geist fonts loaded via next/font/google (self-hosted by Next.js - no external URL)
       - API calls: Supabase (*.supabase.co), Anthropic (api.anthropic.com), Stripe (*.stripe.com)
       - Scripts: No external scripts (all bundled)
       - Styles: No external stylesheets (Tailwind is bundled)
       - Images: No external images currently
       - Connect (XHR/fetch): Supabase, Stripe, Anthropic

    2. **Add CSP header** to next.config.ts headers():
       ```
       Content-Security-Policy:
         default-src 'self';
         script-src 'self' 'unsafe-inline' 'unsafe-eval' https://js.stripe.com;
         style-src 'self' 'unsafe-inline';
         img-src 'self' data: blob:;
         font-src 'self';
         connect-src 'self' https://*.supabase.co wss://*.supabase.co https://api.anthropic.com https://api.stripe.com;
         frame-src https://js.stripe.com https://hooks.stripe.com;
         object-src 'none';
         base-uri 'self';
         form-action 'self';
         frame-ancestors 'none';
         upgrade-insecure-requests;
       ```

    3. **Notes on CSP directives:**
       - script-src 'unsafe-inline' 'unsafe-eval': Required for Next.js development and some Stripe JS
       - style-src 'unsafe-inline': Required for Tailwind JIT and emotion-based libs
       - connect-src includes wss:// for Supabase realtime (if used later)
       - frame-src for Stripe Elements iframe
       - frame-ancestors 'none' matches X-Frame-Options: DENY
       - upgrade-insecure-requests for automatic HTTPS upgrade

    4. **Format as single line** in the header value (newlines shown above for readability)
  </action>
  <verify>npm run build succeeds, curl -I localhost:3000 shows Content-Security-Policy header</verify>
  <done>CSP header protects against XSS while allowing all legitimate resources</done>
</task>
