<task type="auto">
  <name>Task 1: Create Chatbox component with offline awareness</name>
  <files>src/components/calendar/chatbox.tsx</files>
  <action>
Create src/components/calendar/chatbox.tsx with:

1. Props interface:
```typescript
interface ChatboxProps {
  weekStart: Date
  onScheduleChange?: () => void  // Callback when schedule modified via chat
  className?: string
}
```

2. State management:
- messages: Array of { role: 'user' | 'assistant', content: string, timestamp: string }
- inputValue: string
- isLoading: boolean (for typing indicator)
- error: string | null
- llmStatus: { available: boolean, message?: string } | null

3. On mount: Check LLM status via GET /api/llm/status
   - If unavailable, show disabled state with offline message
   - If available, enable chat input

4. **Offline state UI:**
```tsx
{!llmStatus?.available && (
  <div className="flex flex-col items-center justify-center h-full text-muted-foreground p-4 text-center">
    <CloudOff className="h-8 w-8 mb-2" />
    <p className="text-sm font-medium">Chat Unavailable</p>
    <p className="text-xs mt-1">{llmStatus?.message || 'LLM server is offline'}</p>
  </div>
)}
```

5. **Online state UI:**
- Message display:
  - User messages: right-aligned, primary background
  - Assistant messages: left-aligned, muted background
  - Auto-scroll to bottom on new messages
  - Empty state: "Ask me anything about your schedule..."

- Input area:
  - Textarea (auto-grow, max 3 lines)
  - Send button (disabled when empty, loading, or offline)
  - Submit on Enter (Shift+Enter for newline)
  - Clear input on send

- Loading state:
  - Typing indicator (3 dots animation) when waiting for response
  - Disable input while loading

- Error handling:
  - Show error message below input if API fails
  - Handle 503 (offline) specially - update llmStatus

6. Periodic status check: Re-check /api/llm/status every 30s to detect when LLM comes online/offline

Use shadcn Button, existing Textarea from ui/textarea.tsx. Use lucide-react icons (Send, Bot, User, CloudOff).
  </action>
  <verify>npm run build passes, component exports correctly</verify>
  <done>Chatbox component with offline-aware UI</done>
</task>
