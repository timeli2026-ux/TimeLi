# Plan 6.5-01 Summary: LLM Infrastructure

## Overview

**Status:** Complete
**Duration:** ~15 minutes
**Tasks:** 16/16

## What Was Built

### LLM Provider Abstraction
- **Types** (`src/lib/llm/types.ts`): ChatMessage, ChatOptions, ChatResponse, LLMStatus, LLMProvider interface
- **SelfHostedProvider** (`src/lib/llm/providers/self-hosted.ts`): vLLM connection with health checking and 30s caching
- **OpenAIProvider** (`src/lib/llm/providers/openai.ts`): GPT-4o-mini fallback with per-user rate limiting
- **OfflineProvider** (`src/lib/llm/providers/offline.ts`): Graceful degradation with contextual messages

### Hybrid Routing Logic
- **Router** (`src/lib/llm/router.ts`): Time-window aware provider selection
- **Config** (`src/lib/llm/config.ts`): Schedule parsing, timezone support, availability window checking
- Priority: self-hosted (in window) -> API fallback (with quota) -> offline

### Per-User Rate Limiting
- **Service** (`src/lib/services/api-usage.ts`): Track daily API usage per user
- **Migration** (`supabase/migrations/00007_api_usage.sql`): api_usage table with atomic increment function
- **Limit:** 10 messages per user per day when using OpenAI API fallback

### API Endpoints
- **Chat** (`src/app/api/chat/route.ts`): POST /api/chat with conversation persistence, rate limiting
- **Status** (`src/app/api/llm/status/route.ts`): GET /api/llm/status with mode and usage info

### Environment Configuration
- LLM_BASE_URL, LLM_MODEL (self-hosted)
- LLM_SCHEDULE, LLM_TIMEZONE (scheduling)
- OPENAI_API_KEY, OPENAI_MODEL (fallback)

## Architecture Decision

**Hybrid fallback with API rate limit:**
- Self-hosted vLLM runs during scheduled GPU windows (cost-optimized)
- OpenAI API as fallback when self-hosted unavailable (10 msg/day limit)
- Offline mode when both unavailable (limit reached or no config)
