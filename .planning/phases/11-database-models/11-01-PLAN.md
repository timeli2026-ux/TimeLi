  <action>
Create migration file with:

**Table: courses**
- id: uuid primary key (gen_random_uuid())
- user_id: uuid FK to auth.users (cascade delete), not null
- name: text not null (e.g., "CS 301 - Algorithms")
- instructor: text (optional)
- color: text (hex color for calendar display, default '#3B82F6')
- schedule: jsonb not null (array of meeting times: [{day: 0-6, start: "09:00", end: "10:30"}])
- location: text (optional - classroom)
- credits: integer (optional)
- semester: text (e.g., "Spring 2026")
- is_active: boolean default true
- created_at, updated_at: timestamptz

**RLS policies** (follow user-owns-row pattern from life_realms):
- SELECT: auth.uid() = user_id
- INSERT: auth.uid() = user_id
- UPDATE: auth.uid() = user_id (using + with check)
- DELETE: auth.uid() = user_id

**Indexes:**
- courses_user_id_idx on (user_id)
- courses_semester_idx on (user_id, semester)

**Trigger:** updated_at using existing update_updated_at() function
  </action>

**courseScheduleSchema:**
- day: number 0-6 (Sunday-Saturday)
- start: string (HH:mm format)

- name: string, min 1, max 100
- instructor: string optional, max 100

- location: string optional, max 100
- credits: number optional, min 1, max 10
- semester: string, min 1, max 50
- isActive: boolean optional, default true

**courseUpdateSchema:** partial of courseFormSchema

**courseResponseSchema:** courseFormSchema extended with id, userId, createdAt, updatedAt

Export TypeScript types: CourseFormData, CourseUpdateData, CourseResponse, CourseSchedule
